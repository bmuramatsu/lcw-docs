About Learner Credential Wallet

Learner Credential Wallet is a cross-platform iOS and Android mobile application for storing, sharing and mangaging digital learner credentials.

Key features include:

  • Receiving digital credentials from standards compliant issuers via link, QR code, or JSON
  • Displaying and manage digital credentials
  • Sharing one or more credentials as public links or JSON files, e.g., for job applications
  • Adding a digital credential to a LinkedIn Profile
  • Backuping and restoring the wallet
  • Keeping credential access safe with strong encryption best practices

The wallet is based on the Digital Credentials Consortium's learner credential wallet specification and the World Wide Web Consortium's Verifiable Credential Data Model.

Install Learner Credential Wallet

Install Learner Credential Wallet from the Apple App Store or Google Play Store.

Quick Setup

Quick Setup:

After installation, Learner Credential Wallet walks the user through a setup process. Select Quick Setup to continue.

Quick Setup

Quick Setup:

After installation, Learner Credential Wallet walks the user through a setup process. Select Quick Setup to continue.

Create Password:

The user must setup a password to secure the wallet. The user will not be able to recover a lost password, they will need to setup the wallet and request their digital credentials again, or restore from a backup. After entering the requested information select Next to continue.

Register Wallet (in Learner Credential Wallet app):

The user can choose to register the wallet to receive credentials from web applications. Select Go to Register Wallet to register the wallet. Select Next/Skip to skip registering the wallet.

Register Wallet (on Learner Credential Wallet website):

If the user selects Go to Register Wallet, they are taken to the Learner Credential Wallet website to complete the registration processs. Select Register to continue with registration or return to the Learner Credential Wallet app to continue app setup.

Register (on Learner Credential Wallet website):

If the user selects Register, they are asked if they will allow the "Learner Credential Wallet, lcw.app" website to manage credentials for them. Select Next to continue or Cancel to cancel the registration process. If the user choses Cancel they will need to return to the Learner Credential Wallet app to continue app setup.

Allow Wallet (on authn.io website):

If the user selects Next, they are asked to confirm if they will allow the "Learner Credential Wallet, lcw.app" website to manage credentials for them. Select Allow to continue or Block to prevent the "Learner Credential Wallet, lcw.app" website from managing credentials. After completing this step the user will need to return to the Learner Credential Wallet app to continue app setup.

Creating Wallet:

After proceeding from Register Wallet, the app completes the setup process. Select Take Me To My Wallet to continue.

homeHome (Screen)

From the Learner Credential Wallet Home, the user can access the main navigation of the app and view digital credentials that have been added to the wallet.

Main navigation:

  • add_circleAdd Credential(s): First time users should skip to this section to add a credential(s).
  • shareShare Credential(s)
  • settingsSettings

Home provides a list of digital credentials in Learner Credential Wallet. The user can select a credential to view, access shortcuts to manage a credential or add a credential.

Home (with a credential):

Once one or more digital credentials have been added to Learner Credential Wallet, the user can view the list of credentials, select a credential to view and access shortcuts to manage a credential from Home.

For each supported credential, Learner Credential Wallet displays the name of the credential, an image, the issuer's name and verification status.

The Verification Status is displayed as: check_circle (green circle with a check) Verified, close (red x) Not Verified or sync (arrows circling) Verifying.

Supported credentials are re-verified on a regular basis when Learner Credential Wallet has Internet access.

To view a credential select the credential (or the > symbol).

Shortcut to Share a Credential:

From Home, the user can shareShare an individual credential by swiping from the left.

Shortcut to Delete a Credential:

From Home, the user can deleteDelete an individual credential by swiping from the right.

add_circleAdd Credential (Screen)

One can add supported credentials to Learner Credential Wallet in two ways within the app. From homeHome a user can select Add Credential and from any screen the user can select add_circleAdd Credential(s) from the bottom navigation menu. After making this selection the user is presented the Add Credential screen.

Add Credential:

The user can add a supported digital credential using one of the following methods:

  • Scan QR Code: If the user selects this option they are taken to a screen where they can use their mobile device's camera to scan a supported QR code. The QR code may contain one or more credentials, or it may send the user to a website from which they can add one or more credentials.

    If this is the first time using this feature, the user may need to give Learner Credential Wallet permission to use the camera. If the user does not grant permission but wants to scan a QR code at a later time, they can go to their mobile operating system's settings and grant permission for the app to tuse the camera.

  • Add from file: If the user selects this option they are taken to their mobile device's file browser where they can select a supported (JSON) file containing one or more credentials.

  • Add: Paste JSON or URL: The user can paste a supported JSON file or URL to add one or more credentials.

    Adding one or more credentials by URL may add the credential directly from a website or may have the user follow a process for adding the credential which may include authentication and other steps.

Adding a Credential

Adding a credential by any of Add Credential methods, or via a link sent to the user via email or provided on a website, takes the user through a process to add the supported digital credential to their wallet.

Available Credentials:

After selecting one of the methods from the Add Credential Screen, the user is presented a list of supported digital credentials that they may add to their wallet.

The user can choose to Add an individual credential, or if there are more than one credentials they can select Accept All. The user can also choose to preview a credential by selecting the credential (or the > symbol).

If a supported digital credential is already in their wallet, Learner Credential Wallet will ask them if they want to Skip or Accept the credential before proceeding.

Credential Preview:

The user can preview a credential before adding it to their wallet. The user will see the information the issuer of the supported credential included in the credential including a credential name and description.

Learner Credential Wallet displays the Verification Status as: check_circle (green circle with a check) or check (green check) Verified, close (red x) Not Verified or sync (arrows circling) Verifying.

Credential Preview (Issuer Info):

The user can select the Issuer Name or the info symbol to view info on the issuer.

Credential Preview (Verification Status):

The user can select Credential Verified (or the > symbol) to view the Verification Status.

The Verification Status will display the information that Learner Credential Wallet can verify from the registries it is aware of and by examining the cryptographic signatures contained within the supported digital credential. It will also display the last time verification was completed.

Learner Credential Wallet displays the Verification Status as: check_circle (green circle with a check) or check (green check) Verified, close (red x) Not Verified or sync (arrows circling) Verifying.

Supported credentials are re-verified on a regular basis when Learner Credential Wallet has Internet access.

Available Credentials (Added to Wallet):

If the user chooses to add the credential(s) to their wallet, Learner Credential Wallet will display the outcome.

The user should select Done to return homeHome.

Sharing a Credential

shareShare Credential(s) (Screen)

shareShare Credential(s) is available from the main navigation at the bottom of Learner Credential Wallet. After making this selection the user is presented the Share Credential(s) screen.

shareShare Credential(s)

The user can share a supported credential using one of following methods:

  • Create a public link: Allows publicly sharing one credential at a time. The process to create a public link is similar between the main navigation shareShare Credential(s) and the Credential Preview Share menu item. Please see Sharing a Credential for more information.
  • Send a credential: Allows sending one or more credential, typically as an text file.
  • Scan a shared QR code: CHECK/ADD MORE DETAILS Allows sending a shared QR code to send one or more credentials.

Send Credential(s)

Learner Credential Wallet allows the user to Send Credential(s) via the mobile operating systems "Share Sheet". To proceed, the user should can one or more credentials and then select Send Selected Credentials.

Share Preview

Learner Credential Wallet allows the user to preview the credential(s) the user has selected to send. To proceed the select Send.

Share Sheet (iOS / iPadOS)

Learner Credential Wallet displays the mobile operating system "Share Sheet" to allow the user to attach a JSON text file containing the selected credential(s). Typically the user is expected to email, save a note or save the credential)(s) as a file. The user should select the app or menu option to send the credntial(s). After selecting an option the user is returned to the share preview where they can select the arrow_back back arrow to return to the Share screen or select a main navigation option.
