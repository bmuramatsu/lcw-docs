About Learner Credential Wallet

Learner Credential Wallet is a cross-platform iOS and Android mobile application for storing, sharing and mangaging digital learner credentials.

Key features include:

  • Receiving supported credentials from standards compliant issuers via a link, QR code, or JSON
  • Displaying and managing supported credentials
  • Sharing individual credentials as public links and adding a digital credential to a LinkedIn Profile, e.g., for job use on job applications
  • Sharing one or more credentials as file attachments
  • Backuping and restoring the wallet
  • Keeping credential access safe with strong encryption best practices

The wallet is based on the Digital Credentials Consortium's learner credential wallet specification and the World Wide Web Consortium's Verifiable Credential Data Model.

Install

Install Learner Credential Wallet from the Apple App Store or Google Play Store.

Quick Setup

Quick Setup:

After installation, Learner Credential Wallet takes the user through a setup process. Select Quick Setup to continue.

Setup Password:

The user must setup a password to secure the wallet. The user will not be able to recover a lost password, they will need to setup the wallet and request their digital credentials again, or restore from a backup. After entering the requested information select Next to continue.

Register Wallet (in Learner Credential Wallet app):

The user can choose to register the wallet to receive credentials from web applications. Select Go to Register Wallet to register the wallet. Select Next/Skip to proceed without registering the wallet.

Register Wallet (on Learner Credential Wallet website):

If the user selects Go to Register Wallet, they are taken to the Learner Credential Wallet website to complete the registration processs. Select Register to continue with registration, or return to the Learner Credential Wallet app to continue app setup.

Register (on Learner Credential Wallet website):

If the user selects Register, they are asked if they will allow the "Learner Credential Wallet, lcw.app" website to manage credentials for them. Select Next to continue or Cancel to cancel the registration process. If the user choses Cancel they will need to return to the Learner Credential Wallet app to continue app setup.

Allow Wallet (on authn.io website):

If the user selects Next, they are asked to confirm if they will allow the "Learner Credential Wallet, lcw.app" website to manage credentials for them. Select Allow to continue or Block to prevent the "Learner Credential Wallet, lcw.app" website from managing credentials. After completing this step the user will need to return to the Learner Credential Wallet app to continue app setup.

Creating Wallet:

After proceeding from Register Wallet, the app completes the setup process. Select Take Me To My Wallet to continue.

homeHome (Screen)

From the Learner Credential Wallet Home, the user can access the main navigation of the app along the bottom of the screen and view digital credentials that have been added to the wallet.

Main navigation:

  • add_circleAdd Credential(s): First time users should skip to this section to add a credential(s).
  • shareShare Credential(s)
  • settingsSettings

Home provides a list of digital credentials in Learner Credential Wallet. The user can select a credential to view, access shortcuts to manage a credential or Add Credential.

Home (Screen, With a Credential):

Once one or more digital credentials have been added to Learner Credential Wallet, the user can view the list of credentials, select a credential to view and access shortcuts to manage a credential from Home.

For each supported credential, Learner Credential Wallet displays the name of the credential, an image, the issuer's name and verification status.

The Verification Status is displayed as: check_circle (green circle with a check) Verified, close (red x) Not Verified or sync (arrows circling) Verifying.

Supported credentials are re-verified on a regular basis when Learner Credential Wallet has Internet access.

To view a credential select the credential (or the > symbol).

Shortcut to Share a Credential:

From Home, the user can shareShare an individual credential by swiping from the left.

Shortcut to Delete a Credential:

From Home, the user can deleteDelete an individual credential by swiping from the right.

add_circleAdd Credential (Screen)

A user can add supported credentials to Learner Credential Wallet in two ways from within the app. From homeHome a user can select Add Credential and from any screen the user can select add_circleAdd Credential(s) from the main navigation at the bottom of the screen. After making this selection the user is presented the Add Credential screen.

Add Credential (Screen):

The user can add a supported digital credential using one of the following methods:

  • Scan QR Code: If the user selects this option they can use their mobile device's camera to scan a supported QR code. The QR code may contain one or more credentials, or it may send the user to a website from which they can add one or more credentials.

    If this is the first time using this feature, the user may need to give Learner Credential Wallet permission to use the camera. If the user does not grant permission but wants to scan a QR code at a later time, they can go to their mobile operating system's settings and grant permission for Learner Credential Wallet to use the camera.

  • Add from file: If the user selects this option they are taken to their mobile device's file browser where they can select a supported (JSON) file containing one or more credentials.

  • Add: Paste JSON or URL: The user can paste a supported JSON file or URL to add one or more credentials.

    Adding one or more credentials by URL may add the credential directly from a website or may have the user follow a process for adding the credential which may include authentication and other steps.

Adding a Credential

Adding a credential by any of Add Credential methods, or via a link sent to the user via email or provided on a website, takes the user through a process to add the supported digital credential to their wallet.

Available Credentials:

After selecting one of the Add Credential methods, the user is presented a list of supported digital credentials that they may add to their wallet.

The user can Add an individual credential(s) to add, or if there are more than one credential they can select Accept All. The user can also choose to preview a credential by selecting the credential (or the > symbol).

If a supported credential is already in their wallet, Learner Credential Wallet will ask them if they want to Skip or Accept the credential before proceeding.

Credential Preview:

The user can preview a credential before adding it to their wallet. The user will see the information the issuer of the supported credential included in the credential, typically including a credential name and description.

Learner Credential Wallet displays the Verification Status as: check_circle (green circle with a check) or check (green check) Verified, close (red x) Not Verified or sync (arrows circling) Verifying.

To proceed, the user should select the arrow_back back (left) arrow.

Credential Preview (Issuer Info):

The user can select the Issuer Name or the info symbol to view info on the issuer.

To proceed, the user should select the arrow_back back (left) arrow.

Credential Preview (Verification Status):

The user can select Credential Verified (or the > symbol) to view the Verification Status.

The Verification Status will display the information that Learner Credential Wallet can verify from supported registries and by examining the cryptographic signatures contained within the supported credential. It will also display the last time verification was completed.

Learner Credential Wallet displays the Verification Status as: check_circle (green circle with a check) or check (green check) Verified, close (red x) Not Verified or sync (arrows circling) Verifying.

Supported credentials are re-verified on a regular basis when Learner Credential Wallet has Internet access.

To proceed, the user should select the arrow_back back (left) arrow.

Available Credentials (Added to Wallet):

If the user chooses to add the credential(s) to their wallet, Learner Credential Wallet will display the results.

To proceed, the user should select Done.

Credential Preview (Menu)

A user can select the more_vertthree vertical dots next to the Credential Preview label to display a menu of actions they can do with the credential, including:

  • Share
  • View Source
  • Delete

View Source

The user can view the source of the credential in JSON format. The user can select within the credential text to Copy the credential source to the clipboard.

To proceed, the user should select Exit.

Delete

The user can choose to delete a credential by selecting Delete, or they can Cancel to return to the Credential Preview.

Deleting a credential cannot be undone. If the user would need to add the credential again, or restore from a Wallet Backup.

shareShare Credential(s)

  • Scan a shared QR code: CHECK/ADD MORE DETAILS Allows sending a shared QR code to send one or more credentials.

Send Credential(s)

Learner Credential Wallet allows the user to Send Credential(s) via the mobile operating systems "Share Sheet".

To proceed, the user should select one or more credentials and then select Send Selected Credentials.

Share Preview

Learner Credential Wallet allows the user to preview the credential(s) the user has selected to send. To proceed select Send, or select the arrow_back back (left) arrow.

Share Sheet (iOS / iPadOS)

Learner Credential Wallet displays the mobile operating system "Share Sheet" to allow the user to attach a JSON text file containing the selected credential(s). Typically the user is expected to email, save a note or save the credential)(s) as a file. The user should select the app or menu option to send the credential(s).

After selecting an option the user is returned to Share Preview where they can select the arrow_back back (left) arrow to return to Share or select a main navigation option.

settingsSettings (Screen)

Settings (Screen)

Settings provides the user access to options and actions that they can do with their wallet, including:

  • Use biometrics to unlock
  • Dark mode
  • Manage profiles
  • Register wallet
  • Restore wallet
  • Backup wallet
  • Reset wallet
  • About
  • Sign out

Use biometrics to unlock

If the user's mobile device supports biometrics to unlock, the user can enable or disable this feature for Learner Credential Wallet by setting the toggle on (enabled) or off (disabled).

To proceed, the user should select a menu item from the bottom of the screen.

Dark Mode

Learner Credential Wallet supports dark and light modes. The user can enable or disable this for Learner Credential Wallet by setting the toggle on (enabled) or off (disabled). Dark mode disabled is shown in the image.

To proceed, the user should select a menu item from the bottom of the screen.

Manage Profiles

Learner Credential Wallet allows the user to create one or more Profiles in which they can store one or more credentials. When Learner Credential Wallet is setup, a Default Profile is created. The user can:

  • Create New Profile add_circle
  • Add Existing Profile add_circle
  • Lisf of Profiles including the number of credentials stored in each Profile and a more_vert (Profile Menu)

To proceed, the user should select the arrow_back back (left) arrow.

Manage Profiles (Create New Profile)

The user can create a new profile by providing a Profile Name and selecting Create Profile. Or they can Cancel to return to Manage Profiles.

Manage Profiles (Add Existing Profile)

The user can choose to add an existing profile by selecting Scan QR code qr_code_scanner, which opens the mobile devices camera to scan a QR code, or Restore from a file upload_file.

To proceed, the user should select the arrow_back back (left) arrow.

Manage Profiles (Profile Menu)

For each Profile, the user can select the more_vertthree vertical dots to view a menu of actions, including:

  • Rename: The user can rename the profile.
  • Backup: The user can backup the profile. Profile Backup works similarly to Backup Wallet
  • View Source: The user can view the source for the profile and credentials contained within it in JSON format. The user can select within the credential text to Copy the credential source to the clipboard.
  • Delete: The user can delete the profile.

To proceed, the user should select the arrow_back back (left) arrow.

Restore Wallet

The user can restore a wallet from a supported backup file (in JSON format). The user should select Choose a file upload_file to proceed with the restore. The user will be presented a file browser to find and select a supported backup file.

Restore Wallet (Restore Complete)

After selecting a wallet file to restore, Learner Credential Wallet displays the results of successfully imported credentials and duplicate credentials that were ignored. The user can select Details to view the details of the restore.

To proceed, the user should select Close.

Backup Wallet

The user can backup the wallet contents into a file for the user to download by selecting Create Backup upload_file. The user can optionally choose to add password protection.

To proceed, after saving the file, the user should return to Learner Credential Wallet app.

Or the user can choose to Cancel the wallet backup.

Backup Wallet (Backing Up Your Wallet, iOS/iPadOS)

Learner Credential Wallet displays the mobile operating system "Share Sheet" to allow the user to save or send a JSON text file containing the wallet backup.

Typically the user is expected to email, save a note or save a file. The user should select the app or menu option to backup the wallet.

Reset Wallet

Resetting Learner Credential Wallet will delete all of the credentials and profiles in the wallet. This action cannot be undone. To proceed to reset the wallet select Yes, otherwise select Cancel.

If the user selects Yes, they will need to Exit App to Restart Application.

About

About describes Learner Credential Wallet and displays version infomation.

To proceed, the user should select the arrow_back back (left) arrow.

Signout

Selecting Signout immediately signs the user out of Learner Credential Wallet.

The user must enter a password to Unlock Wallet or face Use Biometrics.

On the first attempt to Use Biometrics the user will be prompted to allow Learner Credential Wallet to use the biometric method supported by the mobile device.
