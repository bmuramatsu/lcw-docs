Learner Credential Wallet User Guide

Learner Credential Wallet is a cross-platform iOS and Android mobile application for storing, sharing and mangaging digital learner credentials.

Key features include:

  • Receiving supported credentials from standards compliant issuers via a link, QR code, or JSON text
  • Displaying and managing supported credentials
  • Sharing individual credentials as public links and adding a credential to a LinkedIn Profile, e.g., for use on job applications
  • Sharing one or more credentials as files
  • Backing up and restoring the wallet

Users may also want to check the Frequently Asked Questions.

The wallet is based on the Digital Credentials Consortium's learner credential wallet specification and the World Wide Web Consortium's Verifiable Credential Data Model.

Install

Install Learner Credential Wallet from the Apple App Store or Google Play Store.

Quick Setup

Quick Setup:

After installation, Learner Credential Wallet takes the user through a setup process.

Select Quick Setup to continue.

Password:

The user must setup a password to secure the wallet.

The user will not be able to recover a lost password, if a user loses their password they will need to re-setup the wallet and request their credentials again, or restore from a backup.

After entering the requested information select Next arrow_forward to continue.

Register Wallet (in Learner Credential Wallet app):

The user can choose to register the wallet for easy credential adding in the future to receive credentials from web applications.

Select Go to Register Wallet to register the wallet, or select Next/Skip arrow_forward to continue without registering the wallet.

Register Wallet (on Learner Credential Wallet website):

If the user selects Go to Register Wallet, they are taken to the Learner Credential Wallet website to complete the registration processs.

Select Register to continue with registration, or return to the Learner Credential Wallet app to continue app setup.

Allow Wallet (on Learner Credential Wallet website):

If the user selects Register, they are asked if they will allow the "Learner Credential Wallet, lcw.app" website to manage credentials for them.

Select Next to continue or Cancel to cancel the registration process. If the user choses Cancel they will need to return to the Learner Credential Wallet app to continue app setup.

If the user Cancels but wants to register the wallet at a later time, they can go to settingsSettings from the main navigation at the bottom of the screen and select Register Wallet.

Allow Wallet (on authn.io website):

If the user selects Next, they are asked to confirm if they will allow the "Learner Credential Wallet, lcw.app" website to manage credentials for them.

Select Allow to permit or Block to prevent the "Learner Credential Wallet, lcw.app" website from managing credentials. After completing this step the user will need to return to the Learner Credential Wallet app to continue app setup.

If the user does not grant permission but wants to register the wallet at a later time, they can go to settingsSettings from the main navigation at the bottom of the screen and select Register Wallet.

Creating Wallet:

After proceeding from Register Wallet, the app completes the setup process.

To continue, select Take Me To My Wallet.

homeHome (Screen)

From the Learner Credential Wallet Home, the user can:

  • Add Credential
  • Select a credential to view from a list of credentials that have been added to the wallet, or access shortcuts to manage an individual credential

Home, and every screen, provides access to the main navigation along the bottom of the screen:

  • add_circleAdd Credential(s): First time users should skip to this section to add a credential(s).
  • shareShare Credential(s)
  • settingsSettings

Home (With a Credential):

Once one or more supported credentials have been added to Learner Credential Wallet, the user can view the list of credentials, select a credential to view and access shortcuts to manage a credential from Home.

For each supported credential, Learner Credential Wallet displays the name of the credential, an image and the issuer's name/. The user will also see if Learner Credential Wallet can verify the credential from supported registries and by examining the crypotgraphic signatures contained within the supported credential.

Verification Status is displayed as: check_circle (green circle with a check) Verified, cancel (red x) Invalid Credential or sync (arrows circling) Verifying.

Supported credentials are re-verified on a regular basis when Learner Credential Wallet has Internet access.

To view a credential select the credential (or the > symbol).

Shortcut to Share a Credential:

From Home, the user can shareShare an individual credential by swiping from the left.

Shortcut to Delete a Credential:

From Home, the user can deleteDelete an individual credential by swiping from the right.

add_circleAdd Credential (Screen)

A user can add supported credentials to Learner Credential Wallet in two ways from within the app. From homeHome a user can select Add Credential and from any screen the user can select add_circleAdd Credential(s) from the main navigation at the bottom of the screen. After making this selection the user is presented the Add Credential screen.

Add Credential:

The user can add a supported credential using one of the following methods:

  • Scan QR Code: If the user selects this option they can use their mobile device's camera to scan a supported QR code. The QR code may contain one or more credentials, or it may send the user to a website from which they can add one or more credentials.

    If this is the first time using this feature, the user may need to give Learner Credential Wallet permission to use the camera. If the user does not grant permission but wants to scan a QR code at a later time, they can go to their mobile operating system's settings and grant permission for Learner Credential Wallet to use the camera.

  • Add from file: If the user selects this option they are taken to their mobile device's file browser where they can select a supported JSON text file containing one or more credentials.

  • Add: Paste JSON or URL: The user can paste supported JSON text or a URL to add one or more credentials.

    Adding a credential by URL may add one or more credentials directly from a website, or may have the user follow a process for adding one or more credentials which may include authentication and other steps.

Adding a Credential

Adding a credential by any of Add Credential methods, or via a link sent to the user via email or provided on a website, takes the user through a process to add the supported credential to their wallet.

Available Credentials:

After selecting one of the Add Credential methods, the user is presented a list of supported credentials that they may add to their wallet.

The user can Accept an individual credential, Decline an individual credential, or they can select Accept All. The user can also choose to preview a credential by selecting the credential (or the > symbol).

If a supported credential is already in their wallet, Learner Credential Wallet will ask them if they want to Skip or Accept the credential before proceeding.

Credential Preview:

The user can preview a credential before adding it to their wallet. The user will see the information the issuer of the supported credential included in the credential, typically including a credential name and description. The user will also see if Learner Credential Wallet can verify the credential from supported registries and by examining the crypotgraphic signatures contained within the supported credential.

Learner Credential Wallet displays Verification Status as: check_circle (green circle with a check) Verified, check_circle (green circle with a check) Credential Verified, check (green check) verified, cancel (red x) Invalid Credential, close (red x) not verified or sync (arrows circling) Verifying.

To continue, select the arrow_back back (left) arrow.

Issuer Info:

The user can select the Issuer Name or the info symbol to view info on the issuer.

To continue, select the arrow_back back (left) arrow.

Verification Status:

The user can select check_circleCredential Verified, cancel (red x) Invalid Credential (or the > symbol) to view Verification Status.

Verification Status will display the information that Learner Credential Wallet can verify from supported registries and by examining the cryptographic signatures contained within the supported credential. It will also display the last time verification was completed.

Learner Credential Wallet displays Verification Status as: check_circle (green circle with a check) Verified, check_circle (green circle with a check) Credential Verified, check (green check) verified, cancel (red x) Invalid Credential, close (red x) not verified or sync (arrows circling) Verifying.

Supported credentials are re-verified on a regular basis when Learner Credential Wallet has Internet access.

To continue, select the arrow_back back (left) arrow.

Available Credentials (Added to Wallet):

If the user chooses to add the credential(s) to their wallet, Learner Credential Wallet will display the results.

To continue, select Done.

Create a Public Link and Add to LinkedIn Profile

Create Public Link

Creating a Public Link is a voluntary process. We provide this functionality so the user may choose to share a credential from their wallet to anyone with the link.

Creating a public link will generate a webpage to display the contents of a credential on the VerifierPlus website managed by the Digital Credentials Consortium. The user is fully in control of how and with whom they share this link.

The Public Link is governed by the VerifierPlus Terms and Conditions of Use and Privacy Policy.

To continue, select Create Link, or Cancel.

Public Link

After the Public Link is created, the user can Copy the link to the clipboard, link_off Unshare the link to remove the public link or open_in_newView Link.

The user can also choose to Add to LinkedIn Profile.

To continue, select a Share Credential menu option, select the arrow_back back (left) arrow, or select a main navigation option at the bottom of the screen.

Public Link on VerifierPlus

If the users selects View Link they will see the credential as shared and publicly available on the VerifierPlus website.

The credential contains the same information as displayed in Learner Credential Wallet and includes verification status, last checked date and allows the viewer to autorenewRun Verification to re-verify the credential.

Add to LinkedIn Profile

A user can share their credential on their LinkedIn Profile by using the Add to LinkedIn Profile shortcut.

Note: Before a user can add a Public Link created through the Learner Credential Wallet to their LinkedIn Profile, they must:

  • Either have already created a Public Link or Learner Credential Wallet will create a Public Link at the same time as the user selects Add to LinkedIn Profile.
  • Have the LinkedIn app installed on their mobile device. Note: LinkedIn requires the use of the LinkedIn mobile app and not the website for this shortcut to work.
  • Be logged in to their LinkedIn profile.

Add License or Certification (in LinkedIn app)

If Learner Credential Wallet is able to do so, it will add details from a supported credential to the Add license or certification screen in the LinkedIn mobile app.

To add the license or certification, the user can select Save, or close Cancel.

To conitnue, the user will need to return to the Learner Credential Wallet app.

shareShare Credential(s) (Screen)

shareShare Credential(s) is available from the main navigation at the bottom of the screen. After making this selection the user is presented the Share screen.

Share

The user can share a supported credential using one of following methods:

  • Create a public link: Allows publicly sharing one credential at a time. The process to create a public link is similar between the main navigation shareShare Credential(s) and the Credential Preview Share menu option. Please see Create a Public Link for more information.
  • Send a credential: Allows sending one or more credentials as a JSON text file.
  • Scan a shared QR code: Allows scanning a shared QR code to send one or more credentials.

Send Credential(s)

Learner Credential Wallet allows the user to Send Credential(s) via the mobile operating systems "Share Sheet".

To continue, select one or more credentials and then select Send Selected Credentials, or select the arrow_back back (left) arrow to cancel.

Note: The Send Credential action accessible from the shareShare Credential(s) (Screen) allows the user to select one or more credentials to be sent. Similar sending functionality is available from the Credential Preview menu, however only the credential being previewed can be sent.

ShareSend Preview

Learner Credential Wallet allows the user to preview the credential(s) the user has selected to send.

To continue select Send, select the arrow_back back (left) arrow, or select a main navigation option at the bottom of the screen.

settingsSettings (Screen)

Settings

Settings provides the user access to configure settings and perform actions that they can do with their wallet, including:

  • Use biometrics to unlock
  • Dark mode
  • Manage profiles
  • Register wallet
  • Restore wallet
  • Backup wallet
  • Reset wallet
  • About
  • Sign out

Use Biometrics To Unlock

If the user's mobile device supports biometrics to unlock, the user can enable or disable this feature for Learner Credential Wallet by setting the toggle on (enabled) or off (disabled).

To continue, select a Settings menu option, or select a main navigation option at the bottom of the screen..

Dark Mode (Disabled)

Learner Credential Wallet supports dark and light modes. The user can enable or disable this for Learner Credential Wallet by setting the toggle on (enabled) or off (disabled). Dark mode disabled is shown in the image.

To continue, select a Settings menu option, or select a main navigation option at the bottom of the screen.

Manage Profiles

Learner Credential Wallet allows the user to create one or more Profiles in which they can store one or more credentials. When Learner Credential Wallet is setup, a Default Profile is created. The user can:

  • Create Nnew Pprofile add_circle
  • Add Eexisting Pprofile add_circle
  • View a list of profiles including the number of credentials stored in each Profile

Create New Profile

The user can create a new profile by providing a Profile Name and selecting Create Profile, or they can Cancel.

Add Existing Profile

The user can choose to add an existing profile by selecting Scan QR code qr_code_scanner, which opens the mobile devices camera to scan a QR code, or Restore from a file upload_file.

To continue, if the user does not add a profile, select the arrow_back back (left) arrow, or select a main navigation option at the bottom of the screen.

Manage Profiles (Menu)

For each Profile, the user can select the more_vertthree vertical dots to view a menu, including:

  • Rename: The user can rename the profile.
  • Backup: The user can backup the profile and the credentials contained within it. Profile Backup works similarly to Backup Wallet
  • View Source: The user can view the source for the profile and credentials contained within it in JSON text format. The user can select within the text to Copy the source to the clipboard.
  • Delete: The user can delete the profile.

To continue, select the arrow_back back (left) arrow, or select a main navigation option at the bottom of the screen.

Register Wallet

If the user did not register their wallet to receive credentials from web applications upon setting up Learner Credential Wallet, they can select Register Wallet to do so.

Restore Wallet

The user can restore a wallet from a supported backup file (in JSON text format).

The restore will replace the entire contents of the wallet.

Select Choose a file upload_file to find and select a supported backup file, or select the arrow_back back (left) arrow or select a main navigation option at the bottom of the screen.

Restore Wallet (Restore Complete)

After selecting a backup file to restore, Learner Credential Wallet displays the results of successfully imported credentials and duplicate credentials that were ignored. The user can select Details to view the details of the restore.

To continue, select a Settings menu option, or select a main navigation option at the bottom of the screen.

Backup Wallet

The user can backup the wallet into a file by selecting Create Backup upload_file. The user can optionally choose to add password protection. Or the user can choose to Cancel the backup.

To continue, select a Settings menu option or select a main navigation option at the bottom of the screen.

Backing Up Your Wallet (iOS/iPadOS)

Learner Credential Wallet displays the mobile operating system "Share Sheet" to allow the user to save or send a JSON text file containing the wallet backup.

Typically the user is expected to send the file, save a note or save a file. Select an app or menu option to send or save the file.

Reset Wallet

Resetting Learner Credential Wallet will delete all of the credentials and profiles in the wallet.

This action cannot be undone.

To reset the wallet select Yes, or select Cancel.

If the user selects Yes, they will need to Exit App to restart Learner Credential Wallet.

About

About describes Learner Credential Wallet and displays version infomation.

To continue, select the arrow_back back (left) arrow, or select a main navigation option at the bottom of the screen.

Signout

Selecting Signout immediately signs the user out of Learner Credential Wallet.

Unlock Wallet

Unlock Wallet

The user must enter a password to Unlock Wallet or face Use Biometrics.

On the first attempt to Use Biometrics the user will be prompted to allow Learner Credential Wallet to use the biometric method supported by the mobile device.

Supported Credentials

Learner Credential Wallet supports credentials based on the World Wide Web Consortium's Verifiable Credential Data Model. Additional details can be found in the developer documentation.
